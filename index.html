Staging

The stage or index : what will be included in the next commit.

git add pathspec… = « add pathspec to the staging area ».

Mandatory when adding a new file (untracked).

Add everything to the stage

git add . was not enough before Git 2.0 : it doesn't take into account the working directory, which means deletions are not supported.


Changes not staged for commit:
…
  deleted:    index.html

Untracked files:
…
  home.html

To manage knowns and untracked files, we use -A (--all)


$ git add -A && git status
On branch master
Changes to be committed:
…
  renamed:    index.html -> home.html

See the staged area

Commit the stage


$ git commit

You have to include a message to explain your work:


Capitalized, short (50 chars or less) summary

More detailed explanatory text, if necessary.  Wrap it to about 72
characters or so.  In some contexts, the first line is treated as the
subject of an email and the rest of the text as the body.  The blank
line separating the summary from the body is critical (unless you omit
the body entirely); tools like rebase can get confused if you run the
two together.

Write your commit message in the imperative: "Fix bug" and not
"Fixed bug" or "Fixes bug."  This convention matches up with commit
messages generated by commands like git merge and git revert.

Further paragraphs come after blank lines.


$ git commit -m 'commit message of a lazy developer'

After a commit, the stage is empty.

Why ?

Branches are used to develop features isolated from each others.

By default, la main branch is named master.

One branch by feature

Create a branch

Most of the time, when you create a branch, you immediately want to work on it.


(master) $ git branch feature
(master) $ git checkout feature

(master) $ git checkout -b feature

Solve a conflict

Most of the conflicts are simple to solve.

Here is the workflow:

  1. git status firstly (to know what happened)
  2. Examine the 1st conflicted file (git mergetool)
  3. Fix the conflict
  4. git add to mark the conflict of the file as resolved
  5. If conflicting files remaining, go to (2).
  6. Else, git commit.

Modify

the latest commit

git commit --amend replaces by the current state.

Forgot to add a dependency?


$ git add vendor/scripts/underscore.min.js
$ git commit --amend --no-edit

Forgot to remove a sensitive file?


$ git rm --cached config/database.yml
$ echo config/database.yml >> .gitignore && git add .gitignore
$ git commit --amend --no-edit

Need to update the message ?


$ git commit --amend -m 'New message'

Pro Tips

A file to ignore them all!

We do not want to pollute the repo with libraries, heavy binaries, or with our config.


.gitignore

Atomic commits ?

Do not put anything in your commits...

1 commit = 1 reduced permeter.

To get there, one must masteradd and reset, but also diff, show and obviously commit.

Unstage

Remove a file from the stage : git reset.


$ git reset index.html
Unstaged changes after reset:
M index.html

Cancel the stage :


$ git reset
Unstaged changes after reset:
M index.html
